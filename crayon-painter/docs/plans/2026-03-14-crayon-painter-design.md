#蜡笔画笔 (Crayon Painter) - 设计文档

## 项目概述

一个用真实物件作画的 Web 应用。用户拿起任意有颜色的物品对着摄像头移动，屏幕就会出现蜡笔质感的笔触，颜色直接来自手中的物品。画完后点击颜色可查看物品来源。

## 技术栈

- **前端**: 纯 JavaScript (ES6+)
- **渲染**: HTML5 Canvas 2D API
- **摄像头**: MediaDevices API (getUserMedia)
- **语音**: Web Speech API (语音输入标签)

## 核心模块设计

### 1. CameraManager
- 初始化用户摄像头
- 提供实时视频流
- 暴露视频元素供颜色提取使用

### 2. ColorExtractor
- 从摄像头画面中心区域（取样点）提取颜色
- 使用 Canvas getImageData 获取 RGB 值
- 平滑处理：加权平均最近几帧颜色，避免抖动

### 3. StrokeRenderer (蜡笔质感算法)
- **笔刷形状**: 圆形笔刷 + 边缘噪点
- **多重笔迹**: 每次落笔生成 3-5 个微小偏移的圆形，模拟蜡笔不均匀涂色
- **颜色变化**: 颜色带轻微随机变化（±5% HSL）
- **边缘处理**: 使用随机噪点做粗糙边缘
- **透明度叠加**: 多层叠加产生蜡笔的覆盖感

### 4. CanvasManager
- 画布状态管理
- 记录笔触历史（颜色、位置、时间、标签）
- 提供"点击识别"功能：根据点击位置查找对应的笔触记录

### 5. ObjectLabeler
- 语音输入：用户说出物品名称，使用 Web Speech API 识别
- 预设标签盘：常见物品快捷按钮
- 当前活跃标签状态管理

## 交互流程

1. **启动**: 用户打开页面，点击开始按钮授权摄像头
2. **绘画**: 拿起彩色物品在摄像头前移动，实时生成笔触
3. **标签**: 语音输入或点击按钮设置当前物品标签
4. **查看**: 画完后点击画布任意位置，显示该颜色的物品来源

## 界面布局

- 全屏画布（主体）
- 右下角：摄像头小预览（半透明，150x112px）
- 底部中央：当前颜色指示器 + 标签选择按钮
- 点击画布：弹出来源提示（模态框）

## 验收标准

- [ ] 摄像头成功初始化并显示预览
- [ ] 移动彩色物品时，画布实时出现对应颜色的笔触
- [ ] 笔触具有明显的蜡笔质感（粗糙边缘、纹理感）
- [ ] 可以通过语音或按钮设置物品标签
- [ ] 点击画布某处能正确显示该颜色的物品来源
- [ ] 整体体验流畅，无明显延迟
